# Supplier-Network-Risk-Propagation-Analysis-Platform

# Overview
Modern supply chains operate as highly interconnected systems where suppliers, manufacturers, and logistics networks are closely linked. In such environments, a disruption at a single point—such as a supplier failure, transportation delay, or external risk—can affect multiple connected entities and lead to cascading failures across the entire network.

This project focuses on understanding and analyzing these interconnected risks by modeling the supply chain as a network. By representing suppliers as nodes and their relationships as connections, the system helps in studying how disruptions propagate and impact the overall structure. The goal is to provide meaningful insights that can help in improving supply chain resilience and decision-making.


# Problem Statement
Most traditional supply chain risk management approaches evaluate risks at an individual supplier level. While this helps in identifying local risks, it fails to capture the broader impact of disruptions across the entire network.

Due to this limitation, organizations often struggle to identify critical suppliers whose failure can significantly disrupt the system. There is a need for a system that can analyze interdependencies and provide visibility into how risks spread across multiple levels of the supply chain.


# Objectives
The primary objective of this project is to develop a system that can model and analyze supplier network risks in a structured manner. It aims to identify critical suppliers, simulate disruption propagation, and visualize cascading effects across the network.

Additionally, the project focuses on improving the understanding of supply chain vulnerabilities and supporting proactive risk mitigation strategies through better insights and visualization.


# Tech Stack
The project is developed using Python as the core programming language for data processing and analysis. Streamlit is used to build an interactive and user-friendly frontend interface.

Key libraries include:
- Pandas for data handling and preprocessing  
- NetworkX for network graph modeling and analysis  
- Matplotlib for visualization of results  

GitHub is used for version control and project collaboration.
